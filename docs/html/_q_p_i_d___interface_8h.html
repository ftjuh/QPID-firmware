Detailed Description

Part of the QPID autonomous motor driver project.

The QPID interface provides a generic method for passing messages received by the QPID system down to a specifically adressed subsystem (motor driver, encoder, endswitches). It's the main system's job to receive these messages using any protocol of choice (I2C, ESP-now, bluetooth, SPI...). Messages are composed of a number of bytes with the following offsets:

  • 0x00: overall length of message including this byte and the final checksum byte
  • 0x01: destination, i.e. no. of QPID and its subsystem which is being adressed.
    • bits 0-3: QPID no. This allows for QPID systems with max. 16 QPIDs
    • bits 4-6: subsystem (0 = main system; 1 = motor driver; 2 = encoder; 3 = endstop 1; 4 = endstop 2)
    • bit 7: reserved
  • 0x02: command; specific for each subsystem, no general meaning
  • 0x03: command specific parameters start here
  • 0xnn: checksum byte, nn should be length from 0x00
